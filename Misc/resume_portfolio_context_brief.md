## Caution about framing

Do not present every project as professional production software. Some are best framed as:
- “Built”
- “Prototyped”
- “Developed”
- “Created”
- “Designed”
- “Implemented”

Use stronger language only where supported by the artifact.

Avoid vague phrases like:
- “experienced with data”
- “strong problem solver”
- “familiar with programming”
- “worked with sensors”
- “interested in automation”

Replace them with concrete evidence:
- “Built an ESP32-hosted six-channel monitoring dashboard…”
- “Developed a finite-difference photothermal conversion model…”
- “Created a Python CLI pipeline for `.xyz` profilometry files…”
- “Implemented a local GPU transcription web app using FastAPI and faster-whisper…”
